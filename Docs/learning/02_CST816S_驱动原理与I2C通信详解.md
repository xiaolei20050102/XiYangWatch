# 02 — CST816S 触摸驱动原理与 I2C 通信详解

> 前置知识：已了解 LCD/SPI 驱动原理的读者，本文在对比中展开。

---

## 一、整体图景：手指怎么变成坐标数据？

和 ST7789 一样，CST816S 也是一个独立的芯片，有自己的 MCU 和固件。你只跟它通信，它替你处理触摸检测：

```
┌──────────────┐                 ┌──────────────────┐
│    你的手指   │ ── 电容变化 ──→ │    CST816S 芯片    │
│              │                 │  (内置MCU+固件)    │
└──────────────┘                 │                   │
                                 │  自电容检测算法     │
                                 │  手指坐标计算       │
                                 │  手势识别           │
                                 │  → 结果写入寄存器   │
                                 └────────┬──────────┘
                                          │
                              I2C 总线 (2根线)
                              SDA(数据) + SCL(时钟)
                                          │
                                 ┌────────┴──────────┐
                                 │    STM32F411CE     │
                                 │  读寄存器获取：     │
                                 │  手指数/坐标/手势   │
                                 └───────────────────┘
```

**核心区别：ST7789 是输出设备（你告诉它画什么），CST816S 是输入设备（它告诉你摸哪里）。**

## 二、I2C 通信 —— "共用电话线"

### 2.1 I2C 和 SPI 对比（这是面试高频题）

| 对比项 | SPI | I2C |
|--------|-----|-----|
| **线数** | 4 线（SCK/MOSI/MISO/CS） | **2 线**（SCL 时钟 / SDA 数据） |
| **寻址方式** | 靠 CS 引脚，每个设备单独一根线 | 靠 **地址码**，所有设备共用总线 |
| **速度** | 快（我们 SPI2 = 10.5MHz） | 慢（标准 100kHz，快速 400kHz） |
| **总线挂设备** | 一个 CS 一个设备，多设备多引脚 | 一根总线上挂 **127 个设备** |
| **全双工/半双工** | 全双工（同时收发） | **半双工**（同一时刻只能收或发） |
| **典型用途** | LCD、Flash、高速 ADC | 传感器、触摸、RTC、EEPROM |

**一句话记：SPI 线多但快，I2C 线少但慢。传感器多用 I2C，省引脚。**

### 2.2 I2C 的两根线

| 线名 | 全称 | 作用 |
|------|------|------|
| **SCL** | Serial Clock | 时钟，MCU 发出 |
| **SDA** | Serial Data | 数据线，双向，分时复用 |

两根线都是**开漏输出 + 上拉电阻**。也就是说器件只能拉低，不能主动拉高，靠外部电阻拉到高电平。这样多个设备可以安全地共用一条线而不会短路。

### 2.3 从机地址 —— "叫谁说话"

I2C 总线上每个设备有唯一的 7 位地址。CST816S 的地址是 **0x15**。

```
MCU 发起通信时先发地址：
  ┌──────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬────┐
  │ A[6] │ A[5]│ A[4]│ A[3]│ A[2]│ A[1]│ A[0]│ R/W │    │
  └──────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴────┘
   7-bit 从机地址 = 0x15              方向: 0=写 1=读

HAL 库用 8-bit 地址（左移 1 位）→ 0x2A (写) / 0x2B (读)
```

你的板子上 I2C1 总线上挂的设备：
```
I2C1 (PB6=SCL, PB7=SDA)
├── 0x15  CST816S   触摸
├── 0x6A  QMI8658   姿态传感器（陀螺仪）
├── 0x57  MAX30102  心率血氧
├── 0x76  BME280    温湿气压
└── 0x68  DS3231    时钟芯片
```

**5 个设备共用 2 根线！** 这是 I2C 最大的优势。

### 2.4 I2C 读写时序

**写寄存器：**
```
START → 发地址(写) → 发寄存器号 → 发数据 → STOP
```

**读寄存器：**
```
START → 发地址(写) → 发寄存器号 → RESTART → 发地址(读) → 收数据 → STOP
```

每一步从机回一个 ACK（拉低 SDA 表示收到），如果没收到 ACK 说明设备不在线。

对应代码中的 HAL 宏：

```c
// 写：一次操作完成"地址+寄存器号+数据"
#define TOUCH_I2C_WRITE(reg, pData, sz) \
    HAL_I2C_Mem_Write(TOUCH_I2C, CST816_ADDR_8BIT, (reg), \
                      I2C_MEMADD_SIZE_8BIT, (pData), (sz), 100)

// 读：一次操作完成"地址+寄存器号+地址+收数据"
#define TOUCH_I2C_READ(reg, pData, sz) \
    HAL_I2C_Mem_Read(TOUCH_I2C, CST816_ADDR_8BIT, (reg), \
                     I2C_MEMADD_SIZE_8BIT, (pData), (sz), 100)
```

## 三、CST816S 寄存器模型

### 3.1 关键寄存器

CST816S 也是一套"寄存器模型"——你读写它的寄存器来完成交互，和 ST7789 一样。

| 寄存器地址 | 名称 | 作用 |
|-----------|------|------|
| `0xA7` | ChipID | 芯片 ID，验证通信是否正常 |
| `0x02` | FingerNum | 当前手指数量：0=没摸，0xFF=睡眠 |
| `0x03~0x06` | XposH/L, YposH/L | 触摸坐标（高 4 位 + 低 8 位） |
| `0x01` | GestureID | 硬件手势识别（不可靠，我们不用） |
| `0xE5` | SleepMode | 写 0x03 进入睡眠 |
| `0xF9` | AutoSleepTime | N 秒无触摸后自动低功耗 |

### 3.2 坐标怎么读

坐标存在 4 个寄存器里，X 和 Y 各占 12 位：

```
寄存器 0x03 (XposH): [ x11 x10 x9 x8 | 0 0 0 0 ]   高 4 位
寄存器 0x04 (XposL): [  x7  x6 x5 x4 | x3 x2 x1 x0 ]  低 8 位

X = (XposH & 0x0F) << 8 | XposL   → 范围 0~4095
```

Y 同理。代码就是连续读 4 字节然后拼起来：

```c
void CST816_GetTouch(Touch_Info_t* info)
{
    uint8_t buf[4] = {0};
    TOUCH_I2C_READ(REG_XposH, buf, 4);      // 从 0x03 连续读 4 字节

    info->X_Pos = ((buf[0] & 0x0F) << 8) | buf[1];   // 拼 X
    info->Y_Pos = ((buf[2] & 0x0F) << 8) | buf[3]    // 拼 Y（未加偏移）
                    + TOUCH_OFFSET_Y;                  // 加面板偏移
}
```

### 3.3 TOUCH_OFFSET_Y = 15

和 LCD 的 OFFSET_Y=20 一样，触摸面板也有 bonding 偏移。触摸坐标和显示坐标需要对齐，两个偏移分别配置：

```
手指实际位置 (0, 0) ──触摸面板──→ CST816 输出 (0, 15)
                                         ↓ +TOUCH_OFFSET_Y=15
                                   软件修正为 (0, 0)
```

## 四、手势检测的"正确姿势"

### 4.1 硬件手势寄存器为什么不用

CST816S 有一个硬件手势寄存器 `0x01`，宣称能直接返回 CLICK / DOUBLECLICK / LONGPRESS / UP / DOWN / LEFT / RIGHT。

**实际测试结果：**
- 滑动（UP/DOWN/LEFT/RIGHT）和长按（LONGPRESS）**基本可靠**
- 单机（CLICK）和双机（DOUBLECLICK）**不可靠**——双机几乎检测不到，单机时有时无

**原因：** 手势寄存器读完后会自动清零，但芯片内部判断逻辑不够稳定，固件版本差异大。而且寄存器在一次触摸期间持续报告同一个值，多次读回来需要自己处理去重。

### 4.2 OV-Watch 的正确做法

**根本不用手势寄存器。** 只用两个信息：

```c
CST816_GetFingerNum();   // 手指在不在？
CST816_GetTouch(&info);  // 在哪？
```

手势判断交给应用层或 LVGL，通过**时间差 + 坐标差**来判断：

```
两次按下 < 400ms  →  双击
按住 > 600ms      →  长按
ΔX > ΔY 且 > 阈值 →  左右滑动
ΔY > ΔX 且 > 阈值 →  上下滑动
```

这也是目前我们驱动里的做法——驱动层只提供 PRESS/RELEASE 事件 + 坐标，上层自己判手势。

## 五、初始化流程

参考 [touch_cst816s.c](../BSP/Touch/touch_cst816s.c) 的 `CST816_Init()`：

```
1. 硬件复位：RST 低 10ms → 高 100ms
2. 配置 AutoSleepTime = 5（5 秒无触摸进低功耗）
```

比 ST7789 简单得多，因为触摸芯片不需要配一堆电压/Gamma/时序参数。

## 六、完整交互流程

```
上电
 │
 ├─ CST816_Init()
 │   ├─ RST 低 → 延时 10ms → RST 高 → 延时 100ms (硬件复位)
 │   └─ I2C 写寄存器：AutoSleepTime = 5 (5秒无触摸休眠)
 │
 └─ 循环轮询 (每 10~20ms 一次)
     │
     ├─ I2C 读寄存器 0x02 (FingerNum)
     │   └─ 0x00 = 没摸  /  0x01 = 摸到了  /  0xFF = 在睡觉
     │
     ├─ 如果从 0→1：刚按下 → I2C 读 0x03~0x06 → 打印 PRESS + 坐标
     │
     ├─ 如果从 1→0：刚抬起 → I2C 读 0x03~0x06 → 打印 RELEASE + 坐标
     │
     └─ 应用层根据时间差/坐标差 → 判断 单击/双击/滑动
```

## 七、SPI vs I2C 协议深度对比（面试重点）

| 维度 | SPI | I2C |
|------|-----|-----|
| 信号线 | SCK + MOSI + MISO + CS (4根) | SCL + SDA (2根) |
| 寻址 | 硬件 CS 引脚 | 软件地址码（7-bit） |
| 总线拓扑 | 点对点（一个 CS 管一个设备） | 多主多从（一条总线上百设备） |
| 速度 | 1MHz ~ 50MHz+ | 标准 100kHz / 快速 400kHz |
| 通信模式 | 全双工 | 半双工 |
| 流控 | 无（纯靠时钟） | ACK/NACK 应答 |
| 驱动能力 | 推挽输出 | 开漏 + 上拉 |
| 典型场景 | 高速、点对点（LCD/Flash） | 低速、多设备（传感器） |
| 你的项目 | LCD = SPI2, Flash = SPI1 | 触摸/心率/姿态/温湿压/时钟 = 全部共用 I2C1 |

## 八、你调触摸踩过的坑

| 坑 | 现象 | 原因 | 解决 |
|----|------|------|------|
| **手势寄存器不可靠** | 双击检测不到，单点误报 | CST816S 硬件手势固件不稳定 | 改为只读手指数 + 坐标 |
| **同一手势重复打印** | 一次触摸打印几十条 | 寄存器值在一次触摸期间持续不变 | 加 armed 去重：见 NONE 后才允许再触发 |
| **休眠判据过严** | 自检 FAIL (sleep test) | 部分芯片休眠后 FingerNum 返回 0x00 而非 0xFF | 放宽判据，只要唤醒后能恢复即通过 |

# 面试清单 & 回答要点

---

## Q1：I2C 几根线？和 SPI 有什么区别？

**答：** I2C 只 2 根线（SCL 时钟 + SDA 数据），靠 7-bit 地址码区分设备。SPI 要 4 根（含 CS），靠硬件片选引脚区分。

**关键差异：**
- I2C 总线上可以挂 127 个设备，SPI 每加一个设备多一根 CS 线
- I2C 半双工（收发共用 SDA），SPI 全双工（MOSI 和 MISO 独立）
- I2C 速度慢（100k/400kHz），SPI 快（MHz 级）
- I2C 有 ACK 应答机制，SPI 没有

我项目里 I2C1 上挂了 5 个传感器（触摸/姿态/心率/温湿压/时钟），全部共用 PB6/PB7 两根线，各自有不同地址。

---

## Q2：I2C 地址是什么？7-bit 和 8-bit 地址的关系？

**答：** I2C 标准地址是 7 位的。HAL 库的函数接收 8 位地址，就是把 7 位左移一位（末尾补 0 或 1 表示读/写方向）：

```
CST816S 7-bit 地址 = 0x15 = 0b0001010
HAL 用 8-bit 地址    = 0x15 << 1 = 0x2A = 0b00101010
```

这就是为什么头文件里写 `CST816_ADDR_8BIT = (CST816_ADDR) << 1`。

---

## Q3：CST816S 怎么检测触摸？原理是什么？

**答：** CST816S 用的是**自电容检测**。手指靠近触摸面板时改变了电极和地之间的电容值，芯片内部的检测电路感知到这个电容变化，经过 ADC 采样和算法处理后得出手指坐标。

**重要：** 我们不需要关心电容检测的细节。芯片内部有独立 MCU 和固件，自己完成检测→滤波→坐标计算，结果直接写到寄存器里。我们的 STM32 只通过 I2C 读寄存器，和读一个温度传感器的值一样简单。

---

## Q4：触摸坐标怎么读出来的？

**答：** 连续读 4 个寄存器（0x03~0x06），X 和 Y 各占 12 位：

```
X = (XposH[3:0] << 8) | XposL   // 高 4 位 + 低 8 位
Y = (YposH[3:0] << 8) | YposL + 15  // +TOUCH_OFFSET_Y 对齐显示坐标
```

---

## Q5：你们项目里怎么判断单机、双机、滑动？

**答：** 不用硬件手势寄存器，它不可靠。我们用的是**手指数 + 坐标轮询**方案：

- **按/抬检测：** `CST816_GetFingerNum()` 从 0→1 是按下，1→0 是抬起
- **单机/双机：** 两次按下时间差 < 400ms → 双机，否则单机
- **滑动方向：** 按下的坐标 vs 抬起的坐标，ΔX 和 ΔY 谁大、正负号

手势判断交给应用层，驱动层只提供原始事件。后面接 LVGL 了 LVGL 会替你判断。

---

## Q6：你们 I2C 总线上挂了多少设备？会不会冲突？

**答：** 5 个设备：CST816S(0x15)、QMI8658(0x6A)、MAX30102(0x57)、BME280(0x76)、DS3231(0x68)。

**不会冲突，因为：**
1. 每个设备的 I2C 地址不同（厂商标配，或通过硬件引脚配置）
2. STM32 每次通信先发地址，只有地址匹配的设备才响应
3. 共用同一组引脚 PB6/PB7，HAL 库内部处理了总线仲裁

**但需要注意互斥访问：** 如果 FreeRTOS 两个任务同时读写 I2C1，需要加互斥锁（这是我们后面要加的）。

---

## Q7：触摸模组为什么要硬件复位（RST 引脚）？

**答：**
- 上电后芯片内部状态不确定，需要通过 RST 拉低再拉高让芯片进入已知初始状态
- 休眠唤醒也通过 RST 复位（比写寄存器唤醒更可靠）
- 时序：RST 低 ≥ 10ms → RST 高 ≥ 100ms（等芯片内部初始化完成）

这和 ST7789 的复位时序几乎一样——都是 10ms/100ms。

---

## Q8：如果你在项目中同时用 SPI 和 I2C，会有什么要注意的？

**答：**

| 注意点 | 说明 |
|--------|------|
| **引脚不能冲突** | SPI2 用 PB13/14/15，I2C1 用 PB6/7，互相独立 |
| **中断优先级** | I2C 中断优先级通常低于 SPI（I2C 慢，不会影响 SPI 时序） |
| **RTOS 互斥** | 多个任务访问同一 I2C 总线需要 mutex 保护 |
| **DMA 冲突** | SPI 和 I2C 都用 DMA 时要检查 DMA 通道不冲突 |
| **时序独立性** | SPI 和 I2C 是独立外设，可以同时工作 |

---

## Q9：驱动里面为什么有两层手势函数？后来怎么删了？

**答：** 这是开发过程中的迭代：
1. **v1.0：** 只读硬件手势寄存器 `CST816_GetGesture()`
2. **v1.2：** 发现硬不可靠，加了软件状态机 `CST816_GetEvent()`（300+ 行代码）
3. **v1.3：** 状态机太复杂有 bug，改回纯硬件 + 轻量双击补丁 `CST816_GetGestureEx()`
4. **v1.4：** 参考 OV-Watch，彻底放弃手势寄存器，只用 `GetFingerNum()` + `GetTouch()` 两个函数

**教训：** 不要过度设计。如果芯片自带的功能不靠谱，不要试图在驱动层修补，直接把原始数据交给应用层处理。

---

## Q10：触摸和显示坐标怎么对齐？OFFSET 是什么？

**答：** 两个独立的偏移量：
- **LCD OFFSET_Y = 20：** 显示 bonding 偏移，GRAM 第 20 行才是可见区第一行
- **Touch TOUCH_OFFSET_Y = 15：** 触摸 bonding 偏移，触摸传感器有效区从第 15 行开始

两个值来自模组设计，不同批次可能不同，需要实测校准。对齐到同一个坐标系后，触摸 (x, y) 直接对应显示 (x, y)。
